How do we use it?

So at this point we have Node.js running on our machine and access to all sorts of packages to install. We might have a Sass compiler, a JS file concatinator, a JS minifier, a JS test suite and etc. etc. etc.


How do we use them all in a sane way?

The JavaScript Task Runner


What does it do?

It automates. Performs repetitive tasks like minification, compilation, unit testing, linting, and etc.

How Does it work?

In the root of your project is a file called Gruntfile.js, and in it we define all the tasks you'd like to run.


These tasks include things like grunt build to compile your app for production, or grunt serve --watch to run a local webserver for development.


For our purposes, you can forget we even have a Grunt file. It's been setup already and isn't likely to need changes.

Install every depenancy listed in your bower.json file with:

bower install

All packages are installed in a special folder:

/app/bower_components

This folder is not checked into your git repository.


Unlike it's npm counterpart, this folder is accessable to the web server and the files can be requested from a web browser.

Setting up all these tools is

Painful

Is there an easier way to start a project, give it all these great tools, and a good file structure?

The Web's Scaffolding Tool for Modern Webapps


Yeoman creates new projects, with great tools and great default settings. It provides a great structure for your app to help you be more productive through following best pratices.

Installs and configures your desired tool stack

Select a generator with the stack you'd like to use.



We used a generator named generator-angular-require. It installs and configures all the tools listed above.

Makes creating new apps easy

  1. Install a generator

    npm install -g generator-angular-require

  2. In the directory for your app, run the generator

    yo angular-require {app-name}

Presto you now have an app with all the tools listed on the previous slide (plus a few more) installed, setup and configured with good defaults.

Example: The Route Generator

Generates a controller and view, and configures a route in app/scripts/app.js connecting them.

yo angular-require:route myroute

Produces app/scripts/controllers/myroute.js:


define(['angular'], function (angular) {
  'use strict';
  angular.module('myApp.controllers.myrouteCtrl', [])
    .controller('myrouteCtrl', function ($scope) {
      // ...
    });
});

Produces app/views/myroute.html:


<p>This is the myroute view</p>

Conclusion & Review

Node.js
Allows us to run Javascript from the command line.
npm
Package management for dev suppor tools, supports dependancy management and easy installiation.
Grunt
A task runner that makes all the npm modules sane to deal with.
Karma
Test runner. Runs unit tests with most grunt commands. Errors show up in the console.
Bower
Package management for browser facing tools. Manages dependancies and installing Javascript libs.
Yeoman
Sets up and configures new apps. Provides useful generators to use during development.
